# Project Map — Reference Schema

## Introduction

The **Project Map** is the central contract of the Sequoia framework. It is generated by the context agent (`C0: sequoia-context`) during the `/sequoia init` command and consumed by every phase agent (P1–P6) and meta-agent (M1–M2) to calibrate their analysis.

### When is it generated?

The Project Map is produced at the start of every Sequoia flow:

- `/sequoia init` — generates and displays the Project Map, then stops
- `/sequoia audit` — generates the Project Map as Phase 1, then passes it to all phase agents
- `/sequoia review` — generates a lightweight Project Map scoped to the diff

### Who reads it?

| Consumer | How it uses the Project Map |
|----------|-----------------------------|
| Orchestrator | Decides which phase agents to activate |
| P1–P6 (phase agents) | Calibrate analysis scope and severity thresholds to project type |
| M1 (correlator) | Uses `type` and `stack` to reason about cross-domain root causes |
| M2 (reporter) | Uses `estimated_size` and `indicators` to contextualize the report |

### Chunking behavior

For large projects, generating a complete Project Map in a single context window is not feasible. When the estimated token cost exceeds the threshold (120,000 tokens) OR the project has more than 800 files, the Project Map is generated in **chunks**: named scopes with specific glob patterns, processed in priority order.

## Complete YAML Schema (v0.1)

```yaml
project_map:
  # Schema version — increment on breaking changes
  version: "0.1"

  # --- Identity ---
  name: string                    # Project name (from package.json, go.mod, Cargo.toml, or directory name)
  type: enum                      # frontend | backend | cli | library | fullstack | mobile | infrastructure
  description: string | null      # From package.json description, README first paragraph, or inferred

  # --- Stack ---
  stack:
    languages:                    # All detected languages with approximate percentage of codebase
      - name: string              # e.g. "TypeScript", "Go", "Python"
        percentage: number        # 0–100, all entries sum to ~100
    frameworks:                   # Detected frameworks and libraries with significant architectural role
      - name: string              # e.g. "React", "Gin", "Django"
        version: string | null    # Exact version from lockfile or dependency manifest; null if undetermined
        role: string              # e.g. "frontend", "router", "orm", "testing", "build"
    runtimes:                     # Execution runtimes
      - name: string              # e.g. "Node.js", "Deno", "JVM", "Go runtime"
        version: string | null
    package_managers:             # Detected package managers
      - name: string              # e.g. "npm", "pnpm", "yarn", "pip", "cargo", "go modules"

  # --- Project structure ---
  structure:
    root: string                  # Absolute path to project root
    entry_points: [string]        # Main entry files (e.g. "src/main.ts", "cmd/server/main.go")
    test_dirs: [string]           # Directories containing tests (e.g. "__tests__", "spec/", "internal/*_test.go")
    config_files: [string]        # Key configuration files at root (e.g. ".eslintrc", "go.mod", "pyproject.toml")
    total_files: number           # Total file count (all files, all types)
    total_lines: number           # Approximate total lines of code (source files only)

  # --- Conventions ---
  conventions:
    git_flow: string | null       # Detected branching strategy: "gitflow" | "trunk" | "github-flow" | null
    ci: string | null             # CI provider: "github-actions" | "gitlab-ci" | "circleci" | "none" | null
    linting: [string]             # Active linters (e.g. ["eslint", "stylelint"])
    formatting: [string]          # Active formatters (e.g. ["prettier", "gofmt"])
    testing_framework: string | null  # Primary test framework (e.g. "jest", "vitest", "go test", "pytest")

  # --- Token budget & chunking ---
  token_budget:
    estimated_tokens: number      # Estimated total tokens to represent the full project in context
    chunked: boolean              # true if project exceeds threshold; false if full map fits in one pass
    threshold: number             # Token threshold used to decide chunking (default: 120000)

  # --- Chunks (only present when chunked: true) ---
  # Each chunk defines a named scope processed independently by phase agents
  chunks:
    - name: string                # Human-readable chunk name (e.g. "frontend-components")
      description: string         # What this chunk contains and why it's a logical boundary
      globs: [string]             # Glob patterns for files in this chunk (e.g. ["src/components/**/*"])
      priority: number            # Processing order (1 = first); lower number = processed first
```

## Init Command Logic

### Overview

When `/sequoia init` runs, the context agent (`C0`) performs the following steps:

1. **Scan project files** — Walk the directory tree, collecting all file paths and estimating their token cost based on file type, size, and content complexity.
2. **Estimate token cost** — Aggregate the per-file estimates into a total `estimated_tokens` value.
3. **Decide chunking** — Apply the threshold rule:
   ```
   IF estimated_tokens > 120000 OR total_files > 800:
     chunked = true
     generate chunk definitions
   ELSE:
     chunked = false
     chunks = [] (omitted)
   ```
4. **Generate chunks (if needed)** — Chunk definitions are generated based on `project.type`:

### Chunk Generation by Project Type

```
frontend:
  chunks:
    - name: "components"        globs: ["src/components/**/*"]      priority: 1
    - name: "pages-routes"      globs: ["src/pages/**/*", "src/app/**/*", "src/routes/**/*"]   priority: 2
    - name: "state-hooks"       globs: ["src/hooks/**/*", "src/store/**/*", "src/context/**/*"] priority: 3
    - name: "config-infra"      globs: ["*.config.*", ".github/**/*", "public/**/*"]            priority: 4

backend:
  chunks:
    - name: "domain-services"   globs: ["internal/**/*", "pkg/**/*", "src/services/**/*"]      priority: 1
    - name: "api-handlers"      globs: ["cmd/**/*", "src/routes/**/*", "src/controllers/**/*"] priority: 2
    - name: "data-layer"        globs: ["src/models/**/*", "src/repositories/**/*", "migrations/**/*"] priority: 3
    - name: "config-infra"      globs: ["*.config.*", ".github/**/*", "docker/**/*"]           priority: 4

fullstack:
  chunks:
    - name: "frontend-core"     globs: ["src/components/**/*", "src/pages/**/*", "src/app/**/*"] priority: 1
    - name: "backend-core"      globs: ["server/**/*", "api/**/*", "internal/**/*"]             priority: 2
    - name: "shared-data"       globs: ["src/hooks/**/*", "src/store/**/*", "src/types/**/*"]   priority: 3
    - name: "config-infra"      globs: ["*.config.*", ".github/**/*", "migrations/**/*"]        priority: 4

mobile:
  chunks:
    - name: "screens"           globs: ["src/screens/**/*", "app/**/*"]                        priority: 1
    - name: "components"        globs: ["src/components/**/*"]                                 priority: 2
    - name: "state-services"    globs: ["src/store/**/*", "src/services/**/*"]                 priority: 3
    - name: "config-native"     globs: ["*.config.*", "android/**/*", "ios/**/*"]              priority: 4

infrastructure:
  chunks:
    - name: "core-modules"      globs: ["modules/**/*", "stacks/**/*"]                        priority: 1
    - name: "environments"      globs: ["environments/**/*", "terraform.tfvars*"]              priority: 2
    - name: "pipelines"         globs: [".github/**/*", ".gitlab-ci*", "Makefile"]            priority: 3
```
